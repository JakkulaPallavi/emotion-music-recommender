# Contributing to Emotion-Based Music Recommender

Thank you for your interest in contributing!

## Getting Started

1. Fork the repo and clone your fork
2. Create a virtual environment and install dependencies
3. Create a new branch for your feature or fix

## Pull Request Guidelines

- Write clear commit messages
- Add tests for any new functionality
- Make sure all existing tests pass (`pytest tests/ -v`)
- Update the README if you add new features

## Ideas for Contributions

- More training data (labelled text samples)
- More songs in the database
- New emotions (e.g., disgust, anticipation)
- A web UI (Flask, Streamlit, or FastAPI)
- Spotify API integration
- Better ML models (SVM, Random Forest, BERT)

## Code Style

- Follow PEP 8
- Add docstrings to all functions
- Keep comments beginner-friendly
